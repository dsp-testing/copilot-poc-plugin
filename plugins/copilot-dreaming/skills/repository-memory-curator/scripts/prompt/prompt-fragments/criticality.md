Keep only repository-scoped facts that are **actionable**, **non-obvious**, and **likely to recur** — a good memory should save future agents turns, prevent costly mistakes, or unblock cross-repo work. Drop facts that cost more prompt tokens than they save.

Only a small top slice of the highest-ranked memories (currently ~20–50) is ever injected into an agent's context at once, so the very top of the list must be excellent. Rank by **criticality** = how often a memory helps × how much agent effort it saves × the harm it prevents, and assign each record a coarse **tier** (1 = most critical … 5 = minor). The most actionable, non-obvious, harm-preventing facts are the most critical; obvious or rarely-useful facts rank low or are cut. Be selective: only the highest-ranked records overall — on the order of a few hundred — are ever **stored**, so every fact must earn a scarce slot. A medium-to-large repo should yield **hundreds** of genuinely high-value memories, not thousands of marginal ones — hold the whole set to that bar. This raises the bar for what is worth keeping; it does **not** license cutting a distinct, verified, useful fact, which the verify-and-keep / anti-decimation guard in the curation standards still protects.
